单例模式的几种实现方式

2020-10-12- 2021-04-12
516- 2m
设计模式-单例模式

1、饿汉式

1）特点

  • 在类初始化时直接创建实例对象，不管你是否需要这个对象都会创建
  • 不存在线程安全问题

2）实现思路

  1. 构造器私有化
  2. 自行创建，并用静态变量保存
  3. 向外提供这个实例
  4. 强调这是一个单例，我们可以用final修改

3）三种代码实现方式

a、

public class Singleton1 {
    public static final Singleton1 INSTANCE = new Singleton1();
    private Singleton1(){
    }
}

b、

/**
* 枚举类型，表示该类型的对象是有限的几个
* 我们可以限定1个，就成了单例
*/
public enum Singleton2 {
    INSTANCE
}

c、

/**
* Singleton3与1类似，当需要加载配置文件时才能实例化时使用该种方式
*/
public class Singleton3 {

    public static final Singleton3 INSTANCE;
    private String info;

    static {
        Properties pro = new Properties();
        try {
            /*
            只有资源文件在src下面才能用类加载器
             */
            pro.load(Singleton3.class.getClassLoader().getResourceAsStream("single.properties"));
            INSTANCE = new Singleton3(pro.getProperty("info"));
        } catch (IOException e) {
           throw new RuntimeException(e);
        }
    }

    private Singleton3(String info){
        this.info = info;
    }
     
    //省略getter、setter方法.....

}

2、懒汉式

1）特点：

  • 延迟创建这个实例对象

2）思路

  • 类似同上

3）三种代码实现方式

a、

public class Singleton4 {
    private static  Singleton4 instance;
    private Singleton4(){
    }
    public static Singleton4 getInstance() throws InterruptedException {
        if(instance == null){
            Thread.sleep(1000);
            instance = new Singleton4();
        }
        return instance;
    }
}

b、

/*
  懒汉线程安全式
*/
public class Singleton5 {

    private static volatile Singleton5 instance;//volatile防止指令重排
    private Singleton5(){
    }

    public static Singleton5 getInstance() throws InterruptedException {
        if(instance == null){  //对象已经创建出来后就不用加锁
            synchronized (Singleton5.class){
                if(instance == null){
                    Thread.sleep(1000);
                    instance = new Singleton5();
                }
            }
        }
        return instance;
    }
}

c、

/**
* 在内部类被加载和初始化时，才创建单例对象
* 静态内部类不会自动随着外部类的加载和初始化而初始化，它是要单独去加载和初始化的
* 因为在内部类加载和初始化时创建的，因此是线程安全的
*/
public class Singleton6 {
    private Singleton6(){
    }

    private static class Inner{
        private static final Singleton6 INSTANCE = new Singleton6();
    }

    public static Singleton6 getInstance(){
        return Inner.INSTANCE;
    }
}
