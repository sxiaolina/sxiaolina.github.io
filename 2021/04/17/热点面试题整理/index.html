    Java基础

    String、StringBuffer 和 StringBuilder 的区别是什么? String 为什么是不可变的?

    String为不可变的对象，每次操作都会生成新的对象；StringBuilder和StringBuffer是可变的对象，其中StringBuilder线程不安全，StringBuffer线程安全。

    因为String的底层是一个字节数组，被final修饰，数组一旦创建长度不可变，且由于final修饰，数组的引用不可指向其他对象，因此String不可变。

    StringBuffer/StringBuilder内部实际上是一个字节数组，这个字节数组没有被final修饰，StringBuffer和StringBulider的初始化容量应该为16，当存满之后会进行扩容，底层调用了数组拷贝的方法：System.arraycopy()…扩容的，所以StringBuffer/StringBuilder适用于字符串的频繁拼接操作。

    == 与 equals?hashCode 与 equals ?

    ==和equal都是比较相等的，其中==可以用作基本数据类型和引用数据类型，而equal只可以用作引用数据类型。

    ==在基本数据类型时比较的是值，引用数据类型比较的是内存地址。而equal方法的实现也是利用了==，如Object的equal方法。所以比较的是引用地址。但是有些类重写了equal方法，比较的是对象内的值，如包装类和String类。

    hashCode与equals都是为了比较对象是否相等一致的，hashcode只要比较的是对象的哈希值就可以初步判断对象是否相同，效率比较高，但是不同对象的hash值可能相同，因此还需要用equal方法进一步判断。Java是这样规定的：

    • equal()相等的两个对象他们的hashCode()肯定相等
    • hashCode()相等的两个对象他们的equal()不一定相等

    Java中的集合类往往是根据先判断hashCode，如果相同再调用equal方法进行判断。

    Java 反射？反射有什么缺点？你是怎么理解反射的（为什么框架需要反射）？

    在程序运行期间，对于任何一个类，能获取到这个类的所有方法和属性，还能调用这些方法和属性，这种功能称为反射。

    反射的应用场景了解么？

    框架中也大量使用了动态代理，而动态代理的实现也依赖反射。

    反射机制的优缺点

    优点 ： 可以让咱们的代码更加灵活

    缺点 ：增加了安全问题。比如可以强制获取private属性比如可以无视泛型参数的安全检查。

    获取class对象的四种方式

    知道具体类的情况下可以使用：

    为什么框架需要反射

    反射是框架的灵魂。

    框架可以理解为一个半成品的软件，利用框架进行开发可以大大简化我们的代码。
    其精粹在于框架的设计过程中大量使用了Java的反射机制，将类的各个组成部分封装为其它对象，使得无论你定义了什么类，什么方法，我都可以去使用。

    谈谈对 Java 注解的理解，解决了什么问题？

    一种代码级别的说明。可以声明在包、类、字段、方法、局部变量、方法参数等的前面，用来对这些元素进行说明，注释。

    注解的作用：

    编写文档：通过代码里标识的元数据生成文档【生成文档doc文档】

    代码分析：通过代码里标识的元数据对代码进行分析【使用反射】
    使用反射实现某一种功能

    编译检查：通过代码里标识的元数据让编译器能够实现基本的编译检查【Override】

    Java 泛型了解么？什么是类型擦除？介绍一下常用的通配符？

    泛型是一个未知的数据类型,当我们不确定什么什么数据类型的时候,可以使用泛型

    泛型可以接收任意的数据类型

    创建对象的时候确定泛型的数据类型

    原本的集合类默认都是Object类型的，引入泛型后可以避免类型转换的麻烦，而且如果数据类型插入不正确，编译阶段就能检测出来。

    类型擦除

    泛型信息只存在编译阶段，在进入JVM之前，与泛型相关的信息会被擦除。通俗来讲，泛型类和普通类在java虚拟机内没有什么特别的地方。

    在泛型类被类型擦除的时候，之前泛型类中的类型参数如果没有指定上限，如则会被转成普通的Object类型，如果指定了上限如，则类型参数就会被替换成类型上限

    常用的通配符

    ？ ：代表任意的数据类型；不能创建对象使用；能作为方法的参数使用。

    public static void printArray(ArrayList<?> list){
    //使用迭代器遍历集合
    Iterator<?> it = list.iterator();
    while(it.hasNext()){
        //it.next()方法,取出的元素是Object,可以接收任意的数据类型
        Object o = it.next();
        System.out.println(o);
    }

    泛型的上下限

    泛型的上限限定: ? extends E 代表使用的泛型只能是E类型的子类/本身
    泛型的下限限定: ? super E 代表使用的泛型只能是E类型的父类/本身

    // 泛型的上限：此时的泛型?，必须是Number类型或者Number类型的子类
public static void getElement1(Collection<? extends Number> coll){}
// 泛型的下限：此时的泛型?，必须是Number类型或者Number类型的父类
public static void getElement2(Collection<? super Number> coll){}

    内部类了解吗？匿名内部类了解吗？

    内部类分为成员内部类、静态内部类，方法内部类和匿名内部类。

    内部类可以用 protected、private和static修饰，而外部类只能用public或默认修饰。

    成员内部类

    有一个隐式的外部类的引用，所有可以使用外部类的所有方法和属性。

    外部类.内部类 内部类对象 = new 外部类().new 内部类();
    静态内部类

    没有外部类引用，只能使用外部类的静态属性和静态方法。

    匿名内部类

    没有名字，只能使用一次，在定义后马上使用,它主要是为了重写抽象类和接口中的抽象方法。不能定义构造方法。jdk1.8以后，函数式接口，既可以用匿名内部类实现，也可以用lamda简写。

    BIO,NIO,AIO 有什么区别?

    BIO (Blocking I/O): 同步阻塞 I/O 模式，数据的读取写入必须阻塞在一个线程内等待其完成。

    NIO (Non-blocking/New I/O):是一种同步非阻塞的 I/O 模型，它支持面向缓冲的，基于通道的 I/O 操作方法。

    AIO (Asynchronous I/O):异步非阻塞的 IO 模型。异步 IO 是基于事件和回调机制实现的，也就是应用操作之后会直接返回，不会堵塞在那里，当后台处理完成，操作系统会通知相应的线程进行后续的操作。

    写时复制

    CopyOnWrite容器即写时复制的容器。往一个容器添加元素的时候，不直接往当前容器Object[]添加，而是先将当前容器Object[]进行Copy，复制出一个新的容器Object[] newElements，然后向新的容器Object[] newElements里添加元素。添加元素后，再将原容器的引用指向新的容器setArray(newElements)。这样做的好处是可以对CopyOnWrite容器进行并发的读，而不需要加锁，因为当前容器不会添加任何元素。所以CopyOnWrite容器也是一种读写分离的思想，读和写不同的容器。

    JVM

    jvm内存结构

    分为线程公有和线程私有

    线程公有

    堆，虚拟机启动时创建，作用：存放实例对象，几乎所有的对象和数组都在这里分配内存。

    方法区：存储虚拟机加载的类学习，常量、静态变量。jdk1.7方法区是实现是永久代，jdk1.8后是元空间，且移动到了直接内存中。

    线程私有

    程序计数器：当前线程执行字节码的行号指示器，用来选取下一条指令。

    虚拟机栈：堆管存储，栈管运行；虚拟机栈就是用来存储方法运行时的

    局部变量表，操作数栈等运行数据。存储的数据类型有八大基本数据类型和引用类型。

    本地方法栈：和虚拟机栈类似，不过是为本地方法服务的。

    JVM调优参数

    常见参数

    • -Xms :初始大小内存，默认为物理内存1/64<=>-XX:InitialHeapSize

    • -Xmx：最大分配内存，默认为物理内存1/4<=>-XX:MaxHeapSize

    • -Xss：设置单个线程的大小，一般默认为512K~1024K<=>-XX:ThreadStackSize

    • -Xmn：设置年轻代大小

    • -XX:MetaspaceSize 设置元空间大小

    • -XX:MaxTenuringThreshold：设置垃圾最大年龄

    • -XX:+PrintGCDetails：输出详细GC收集日志信息

    -XX:NewRatio：配置年轻代与老年代在堆的比例：默认 =2 ：新生代占1/3

    -XX:SurvivoRatio：设置新生代占eden和s0/s1空间的比例：默认 = 8，即8:1:1
